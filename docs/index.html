Getting Started

These instructions will get you a copy of the project up and running on your local machine for development and testing purposes.

Prerequisites

You will need both Node.js and Git to be installed on your machines before continuing.

Installation

To install the project, go to the desired folder and open the command line/terminal. Type the command;

git clone https://github.com/samsam-026/openclassrooms-todo-app.git

Once cloned into the machine, go into the newly created folder openclassrooms-todo-app and open the index.html file to use the project.

Testing

To test the repo, first clone it as seen above, then move into the folder from the terminal;

cd openclassrooms-todo-app

Then install the testing dependencies using this command;

npm install

Upon successful installation, the tests can be run when the test/SpecRunner.html file is opened.

MVC Architecture

The architecture used in this project is MVC (Model-View-Controller) that separates an application into 3 main parts. Its is the most commonly used architecture for websites.

Todo Application Flow

The complete flow of the application is illustrated below;

Model

The model is the data related logic of the application.
In this application, the data objects are the todo objects and the list of todos.
Below are the methods implemented for the model;

create(title, callback)

Creates a new todo.

Parameters
  • title: Name of the new todo. (String)
  • callback: Callback to be fired after todo creation. (Function)
read(query, callback)

Return a list of todo according to the chosen filter (All/Active/Completed) or data it contains (id or name). If no query is given, it will return the whole list.

Parameters
  • query: Query to find todos that match. (String | Number | Object)
  • callback: Callback to be fired after todo(s) are found. (Function)
update(id, data, callback)

Update the details of a todo item (todo name/completed or not).

Parameters
  • id: ID of the todo to be found. (Number)
  • data: New values of the specific todo to update. (Object)
  • callback: Callback to be fired after the todo is updated. (Function)
remove(id, callback)

Remove a specific todo using its unique ID.

Parameters
  • id: ID of the todo to be removed. (Number)
  • callback: Callback to be fired after the todo is removed. (Function)
removeAll(callback)

Delete all todos.

Parameters
  • callback: Callback to be fired after all todos are removed. (Function)
getCount(callback)

Get the number of todos according to chosen filter (All/Active/Completed).

Parameters
  • callback: Callback to be fired after todo count are found. (Function)

View

The view contains the UI of the application, which is what the is displayed to the user.
In the case of this application, it is the index.html page.
Below are the methods implemented for the view;

showEntries(todos)

Display all todos as a list on the webpage.

Parameters
  • todos: Todo list to be displayed. (Object)
removeItem(id)

Remove list item li with the corresponding data-id attribute.

Parameters
  • id: data-id attribute value of the HTML li element to be removed. (String)
updateElementCount(activeTodos)

Update the counter that shows the number of active todo items on the webpage.

Parameters
  • activeTodos: Todo list count to be displayed. (Number)
clearCompletedButton(completedCount, visible)

Display the "Clear completed" button if there are completed todos.

Parameters
  • completedCount: Number of completed todos. (Number)
  • visible: Boolean for if button should be displayed. (Boolean)
contentBlockVisibility(visible)

Display the footer which contains all buttons and filters if there is more than one todo.

Parameters
  • visible: Boolean for if footer should be displayed. (Boolean)
toggleAll(checked)

Display a more opaque icon on the "toggle all" button next to the text input, when clicked.

Parameters
  • checked: Boolean for if toggle all checkbox should be selected. (Boolean)
setFilter(currentPage)

Set the selected class on the selected filter (All/Active/Completed).

Parameters
  • currentPage: Filter name "active", "completed" or empty string "" for all. (String)
clearNewTodo()

Clear the value in the text input once a new todo is created.

Parameters
  • N/A
elementCompleted(id, completed)

Toggle the class completed on the todo list item to change its opacity. Additionally, toggle the checked value of the checkbox input of the todo list item.

Parameters
  • id: data-id attribute value of the HTML li element to be completed. (String)
  • completed: Boolean for if li element should have class completed. (Boolean)
editItem(id, title)

Display a text input on the todo list item li when the item is double-clicked.

Parameters
  • id: data-id attribute value of the HTML li element to be edited. (String)
  • title: Current name of todo. (String)
editItemDone(id, title)

Hide the text input once the todo item has been edited and replace the old todo name with the newly entered one.

Parameters
  • id: data-id attribute value of the HTML li element to complete editing. (String)
  • title: Updated name of todo. (String)

Controller

The controller of an application acts as an interface between the model and the view.
It handles the requests made from the user in the view and manipulates the model accordingly.
Below are the methods implemented for the controller;

setView(locationHash)

Triggered any filter button is clicked or on page load.
It handles the subsequent calls to methods showAll, showActive and showCompleted.

Parameters
  • locationHash: Filter name "active", "completed" or empty string "" for all. (String)
showAll()

Triggered when filter button "All" is clicked.
It calls the model method read to get all todos.
Once the todo list is returned, the view methods showEntries, setFilter and updateElementCount are run.

Parameters
  • N/A
showActive()

Triggered when filter button "Active" is clicked.
It calls the model method read to get all active todos.
Once the todo list is returned, the view methods showEntries, setFilter and updateElementCount are run.

Parameters
  • N/A
showCompleted()

Triggered when filter button "Completed" is clicked.
It calls the model method read to get all completed todos.
Once the todo list is returned, the view methods showEntries, setFilter and updateElementCount are run.

Parameters
  • N/A
addItem(title)

Triggered by the typing the enter key in the text input.
It calls the model method create to create a new todo.
Once the todo is created, the view methods showEntries, setFilter and updateElementCount are run.

Parameters
  • title: Name of new todo. (String)
editItem(id)

Triggered when a todo list item is clicked.
It calls the model method read to find the current todo list item name.
The view method editItem is run.

Parameters
  • id: ID of the todo to be edited. (Number)
editItemSave(id, title)

Triggered when the enter key is typed in a text input in a todo list item.
It calls the model method update to replace the new name of the todo.
If an empty string is passed, the model method remove is called instead.
Once the value is updated, the view methods editItemDone, updateElementCount and clearCompletedButton is run.

Parameters
  • id: ID of the todo to complete editing. (Number)
  • title: Updated name of the todo. (String)
editItemCancel(id)

Triggered by clicking outside the text input in a todo list item after it is displayed.
It calls the model method read to find the current todo list item name.
The view method editItemDone is run.

Parameters
  • id: ID of the todo to cancel editing. (Number)
removeItem(id)

Triggered by clicking the cross icon on the right of a todo list item.
It calls the model method remove on the clicked todo to delete it from the list.
Once it is deleted, the view methods updateElementCount and removeItem are run

Parameters
  • id: ID of the todo to be removed. (Number)
toggleComplete(id, completed, silent, checkbox)

Triggered by clicking the checkbox on the left of a todo list item.
It calls the model method update on the clicked todo to make it completed.
Once the todo is updated, the view methods updateElementCount, clearCompletedButton and elementCompleted are run.

Parameters
  • id: ID of the todo to be completed. (Number)
  • completed: Boolean for if all todos are to be completed or active. (Boolean)
  • silent: Boolean for if todo list needs to be re-filtered. (Boolean)
  • checkbox: Checkbox element to display the checked state. (Object)
removeCompletedItems()

Triggered by clicking the "Clear completed" button.
It calls the model method remove on all todos which are completed.
Once the value is updated, the view method removeItem is run for each completed todo.

Parameters
  • N/A
toggleAll(completed)

Triggered when clicking the "toggle all" checkbox next to the text input.
It calls the model method update on each todo to have the same completed state.
Once the value is updated, the view methods toggleAll, clearCompletedButton and elementCompleted are run.

Parameters
  • completed: Boolean for if all todos are to be completed or active. (Boolean)

Store

The store of this application manages the data store, which handles the data persistance to the local storage.
Below are the methods implemented for the store;

save(updateData, callback, id)

Saves either new todo objects or updates the existing one in the store.

Parameters
  • updateData: Todo data including name and completed. (Object)
  • callback: Callback to be fired after the todo is saved/updated. (Function)
  • id: ID of a todo. Will only be used if an original todo is being updated. (Number)
find(query, callback)

Find a todo in the store todo list based on a query.

Parameters
  • query: Query to find todos that match. (String | Number | Object)
  • callback: Callback to be fired after todo(s) are found. (Function)
findAll(callback)

Returns the whole list from the store.

Parameters
  • callback: Callback to be fired after whole todo list is returned. (Function)
remove(id, callback)

Removes a todo object from the list in the store.

Parameters
  • id: ID of the todo to be removed. (Number)
  • callback: Callback to be fired the todo list is removed. (Function)
drop(callback)

Completely clear store.

Parameters
  • callback: Callback to be fired after whole store is cleared. (Function)

Usage

Creating a todo

  1. Click on the text input.
  2. Type in the todo name.
  3. Press the enter key to save the todo.

Model methods

create, getCount

View methods

showEntries, setFilter, updateElementCount, clearNewTodo

Controller methods

addItem

Store methods

save

Updating a todo

  1. Double-click the todo list item.
  2. Type in your new todo name.
  3. Press the enter key to save the todo, OR
  4. Click outside the input to cancel the change.

Model methods

read, update, remove

View methods

editItem, editItemDone, updateElementCount, clearCompletedButton

Controller methods

editItem, editItemSave, editItemCancel

Store methods

save, find

Completing a todo

Click on the checkbox on the left of the list item.

Model methods

update, getCount

View methods

elementCompleted, updateElementCount, clearCompletedButton

Controller methods

toggleComplete

Store methods

save

Removing a todo

Click on the cross icon on the right of the list item.

Model methods

remove, getCount

View methods

removeItem, updateElementCount, clearCompletedButton

Controller methods

removeItem

Store methods

remove

Complete all todos

Click on the checkbox on the left of the text input to toggle all todos complete/active.

Model methods

read, update, getCount

View methods

toggleAll, clearCompletedButton, elementCompleted, updateElementCount

Controller methods

toggleAll

Store methods

save, find

Clear Completed todos

Click on the "Clear completed" button on the right of the footer to delete all completed todos.

Model methods

read, remove

View methods

removeItem, clearCompletedButton, contentBlockVisibility, updateElementCount

Controller methods

removeItem

Store methods

remove, find

Filtering the todo list

  • All: The first view displaying all todos.
  • Active: The second view displaying all non-completed (active) todos only.
  • Completed: The third view displaying all completed todos only.

Model methods

read, getCount

View methods

showEntries

Controller methods

N/A

Store methods

find
